03 — EXISTING STRUCTURE

How the existing builder was structured

Fieldproxy enables organisations to coordinate field teams through mobile apps used by field technicians and web apps used by operations managers.

Data Source
Node Based Builder Interface
Admin Dashboard & Operations Web Portal

The platform used a drag-and-drop builder for web application creation, where implementation teams configured applications, dashboards, and reports using widgets. These were used by operations teams to manage field operations.

Each component required configuration across UI, data, and actions, forcing teams into rigid structures and limiting flexibility.

04 — PROBLEM

What led to the platform rebuild

As Fieldproxy scaled to support larger operations, the existing V3 workflow builder struggled with increasing complexity. Implementation teams found it difficult to configure, debug, and maintain workflows, exposing the key limitations outlined below.

warning

Rigid System Architecture

Heavy reliance on manual configuration and predefined components made the system rigid, limiting flexibility and making it hard to scale across complex use cases.

bolt

Complex Workflow

Multi-layered configuration across UI, data, and actions required navigating several steps, making even simple workflows complex to set up and slowing deployment.

sync

Inconsistent Builder Models

Mobile used a node-based builder while web used a drag-and-drop interface, leading to inconsistent setups and making debugging and stability harder to manage.

This led to higher churn, increased support costs, and slower adoption, impacting end-user experience.

05 — SCALING

Scaling to Web & Introducing AI

After validating the new architecture with the mobile workflow, the next step was to scale the builder to support web applications, which required handling greater layout complexity, larger screen structures, and deeper element hierarchies.

Constraints:

layers

Deep nesting

Manual screen creation became difficult with deep element, component, and layout nesting.

grid_view

Complex Layouts

Web applications required flexible, multi-layered layouts, which increased effort and complexity in building.

deployed_code

Slow Deployment

Large workflows and deep nesting increased time and effort, reducing development velocity at web scale.

These constraints increased effort, slowed deployment, and made it harder to scale with manual workflows. To address this, we introduced an AI-assisted workflow.

 07 — AI WORKFLOW

AI-assisted workflow

A structured generation approach to accelerate deployment and reduce manual effort.

To make the workflow scalable, I defined a structured AI generation system that translated requirements into JSON configurations using reusable elements, layout rules, and screen-level instructions. This workflow was implemented using Cursor, where the design system and generation rules were provided as context. Based on structured prompts, the AI generated valid configurations, which were then integrated into the builder.

Output Generation

With this structured generation workflow, the generator followed a fixed hierarchy to ensure UI, data, and queries were created in the correct order before the configuration was used in the builder. This reduced manual setup, deployment time and made the platform easier to scale for larger applications.

I defined each component with rules, usage guidelines, JSON structure, and supported actions to guide generation.

08 — USER INTERFACE 

The rebuilt platform in action

Every interaction was designed to simplify complex workflows. Below is a comprehensive look at the new system architecture and user experiences across the platform.

SQL Editor - Addition of data source

The workflow begins with the addition of a data source, through schema creation in the SQL Builder, where tables are defined directly using queries. By adopting a data-first workflow, queries and relationships are configured upfront, allowing screens to be generated from structured data rather than manual setup.

The system supports this with structured JSON outputs and table-level previews for validation, along with clear error states to catch issues early in the configuration process.

Empty State

Users begin with a clean, intuitive interface. The builder presents a clear entry point with minimal cognitive load—ready to create their first table and define data structure.

Table Added

Once a table is created, the builder displays the structure. Users can see columns, data types, and available actions. The interface organizes complexity into digestible components.

Data Inserted

Values are populated into the table. The builder now shows a complete data structure with sample data, validating the design and giving users confidence in their workflow definition.

Builder

Introduced a unified builder for creating and managing both web and mobile applications, combining configuration and real-time preview in a single interface. With the shift to a structured generation workflow, screens are created through JSON instead of manual assembly, reducing complexity and removing unnecessary steps.

This streamlined approach enables teams to efficiently build and manage large-scale applications while maintaining consistency and scalability across platforms.

Key Learnings

What I Learned Rebuilding This Platform

Designing and rebuilding this platform taught me that scalability is driven by strong system structure and configuration design.

Moving from an unstructured builder to a more defined model improved stability, maintainability, and implementation speed.

Simplifying complex workflows for internal users reduced manual effort and accelerated development cycles, directly impacting business outcomes. I also designed a new set of widgets and experiences across mobile and web, supported by a unified design system that ensured consistency while enabling flexibility for diverse use cases.

Integrating AI-assisted workflows further highlighted how automation can improve efficiency when applied thoughtfully. This project reinforced the importance of balancing flexibility with structure, and the value of close collaboration with engineering to deliver scalable, real-world solutions.
