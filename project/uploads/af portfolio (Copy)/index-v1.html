§ 02 — How I ship

AI is not a gimmick. It's the loop.

Brief → prototype → critique → ship, compressed from weeks to days. I use AI to type less, decide more, and put real artifacts in front of stakeholders on day one — not month three.

01 · Frame

Brief

I write the problem in one paragraph, the user in one sentence, the success metric in one number.

Memo · Loom · Spec
02 · Diverge

AI

I pair with models on competitive teardown, edge cases, copy variants, JSON schemas, and throw-away code.

Claude · Cursor · v0
03 · Make

Prototype

Real, clickable, in the actual stack. Stakeholders react to artifacts, not screenshots. Decisions accelerate.

Figma · React · Live data
04 · Land

Ship

Cut what doesn't matter. Hand off in one doc. Measure adoption in the week, not the quarter.

PR · Loom · Metric
What this changes
Decisions happen on real artifacts, not slideware.
Where it shows
Field Proxy rebuild shipped in a quarter, not a year.
What I will not do

§ 03 — How I work

Six things you can count on me for.

Not a list of skills. A list of behaviours your PM, CTO and account team will actually feel inside two weeks.

01

Independent, decisive.

Small surface area, fewer meetings. I'll find the work, scope it, and bring options back the same week.

02

Influences direction.

I sit alongside PM and CTO, not below them. I'll push back on the roadmap when the data says so.

03

Goes deep technically.

I learned the JSON schema for the platform rebuild so I'd know what was actually possible to ship.

04

Real enterprise research.

I run sessions with the buyers and the field — both sides, same week. Decisions land because both sides feel heard.

05

Balances stakeholders.

Enterprise customization vs. SaaS sanity. Ops manager vs. field tech. I keep both stories alive in the design.

06

Systems that scale.

A drag-and-drop tool became a JSON system. A dense table became a live map. The right primitive does the heavy lifting.
